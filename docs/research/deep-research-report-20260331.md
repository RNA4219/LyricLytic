# エグゼクティブサマリ

AI音楽生成サービス（Suno/Udio等）を使って歌詞から制作を行うクリエーター向けに、歌詞作成ワークフロー全体を支援するデスクトップ「作詞ワークベンチ」を設計します。本システムは、Microsoft製のオープンソース・コードエディタである**Monaco Editor**（VS Codeをベース）を本文編集エンジンに採用し【24†L78-L80】、Tauriをラッパーとしてデスクトップ化します。TauriはOS標準のWebViewを用いるためElectronより脆弱性リスクが低く、厳格なCSP設定や最小限の権限付与によりXSS攻撃などを抑制できます【26†L9-L14】【4†L184-L190】。MVPでは、主に**歌詞の断片収集・編集・推敲・曲との紐付け・クリップボード出力・履歴管理**など、制作ワークフローの骨格機能に注力し、AI自動生成や専用スマホアプリは後回しとします。これにより、個人クリエーターが試行錯誤しながら効率的に作詞できる環境を実現します。

## 前提・目的

本プロダクトは、**SunoやUdioなどのAI音楽生成サービスで楽曲を作る個人クリエーター**を対象とします。これらのユーザーは、AIに歌詞を与えて曲を生成する際、外部で歌詞を試行錯誤したり、何度も貼り付け直して調整する必要があります。そのため、以下を目的とします。

- **作詞ワークフロー支援**：デスクトップ上で歌詞を編集・整理し、生成した曲との対応を管理できる環境を提供。  
- **バージョン管理**：試行錯誤の過程で生じる歌詞の改稿履歴を管理し、過去バージョンへの復元や差分比較を可能にする。  
- **高速な入力・出力**：作成した歌詞を即座にクリップボードにコピーし、Suno/Udio等に貼り付けられるようにする。  
- **外部連携のシンプル化**：スマホメモや音声メモなどで採集した歌詞断片のインポート機能を備え（MVPでは簡易版）、柔軟に素材を集約する。  

この設計により、「作詞のたびにUIに不満を感じる」「生成と編集を何度も繰り返す」「過去に良い歌詞があったのに気づけない」といった課題を解消することを目指します。

## 主要ユースケースと画面フロー

作詞ワークベンチでは、次のようなユーザー行動を想定します。

- **採集 (Collect)**: 外出中や就寝前などに浮かんだフレーズや単語を保存する。MVPではPC版テキスト入力やSNS経由で断片を取り込む方法に留め、必要に応じてスマホ連携（後回し）で補強。  
  - *優先度：高 (MVPで簡易対応可)*

- **整形 (Structure)**: 椅子に座って、断片を並べて歌詞の構成（Aメロ/サビ等）を固める。中央エディタで歌詞テキストを編集する段階。  
  - *優先度：高 (MVPで必須)*

- **推敲 (Revise)**: 編集した歌詞案を声に出すなどして読み味を確認し、母音・韻・凡庸表現を修正する。編集画面で差分表示や凡庸語検知などの補助情報を参照。  
  - *優先度：高 (MVPで必須)*

- **曲紐付け (Link Song)**: 現在の歌詞版を用いてAI生成した曲を登録し、歌詞と曲の対応を記録・管理する。曲URLやローカルファイルパスと歌詞版を関連付ける。  
  - *優先度：高 (MVPで必須)*

- **コピー (Copy)**: 最終的な歌詞案をクリップボードに即時コピーし、外部のSuno/Udio等サービスに貼り付けられるようにする。全歌詞・セクション単位のコピーをサポート。  
  - *優先度：高 (MVPで必須)*

- **履歴復元 (History & Revert)**: 過去に保存した歌詞版にいつでも戻れるようにし、改稿前後の差分比較も可能にする。  
  - *優先度：高 (MVPで必須)*

画面フローをMermaidで示すと以下のようになります（各ノードはMVPでの重点フェーズ）：

```mermaid
flowchart LR
    A[採集 (Collect)] --> B[整形 (Structure)]
    B --> C[推敲 (Revise)]
    C --> D[曲紐付け (Link Song)]
    D --> E[コピー (Copy)]
    E --> F[履歴復元 (History)]
```  

各ユースケースは高優先度であり、特に「整形」「推敲」「曲紐付け」「コピー」「履歴復元」はMVPに必須です。採集は主に外部で行う性質が強いため、アプリ内では断片の取り込み・表示機能にとどめ、スマホアプリ開発は二次フェーズとします。

## 必須機能 / 非必須機能

**必須機能（MVPで実装）**

- **歌詞全文クリップボードコピー**: 全文・セクション単位でコピー可能。TauriのクリップボードAPI等を利用し、外部サービスへの貼り付けを容易にする。  
- **バージョン管理 (Version Control)**: 任意タイミングで歌詞版をスナップショットとして保存。差分比較や履歴リストから復元できる。  
- **差分比較ビューワ**: 選択した2版間の差分をエディタ内で可視化する（MonacoのDiff Editor機能を利用）【24†L93-L99】。  
- **歌詞–曲データ紐付け**: 歌詞版と生成曲のURL/ファイルを関連付け。曲ごとのメタ情報をプロジェクトに格納。  
- **断片インポート**: 外部メモ（テキスト）からの断片取り込み機能。コピー&ペーストやファイル読み込みで断片リストに追加。  
- **プロジェクト・ブラウズUI**: 左ペインにプロジェクト一覧と歌詞版ツリーを表示し、編集中の歌詞版を切り替えられる。  
- **歌詞エディタ**: 中央ペインにMonacoエディタを埋め込み、キーボード/ショートカット操作をサポート。  
- **コピー・導線最適化**: 編集完了後ワンクリックでコピーできるUIやショートカットを用意。

**非必須機能（MVP後回し）**

- **スマホアプリ版**: iOS/Androidクライアント。まずはPC主体。  
- **高度なAI自動作詞**: LLMベースの自動歌詞生成やリライト機能。（MVPは手動入力重視）  
- **高度なテキスト解析**: 自然言語処理による韻検出、構文評価などは今後拡張。  
- **共同編集/クラウド同期**: 複数端末や複数人での同時編集機能は初期フェーズでは不要。  
- **テーマ/デザイン設定**: UIテーマ切り替え等のカスタマイズは後回し。

## Tauri採用時の禁止事項（要件）

Tauriのセキュリティガイドラインに基づき、本アプリ開発では以下の条件を遵守します。いずれもドキュメント化し、コードレビューで確認します【4†L184-L190】【26†L9-L14】。

- **外部コンテンツロード禁止**: CDN経由や外部サーバーのスクリプト/HTML読込を行わず、**バンドルされたローカル資産のみ**を読み込むこと【4†L184-L190】。CSP設定は`default-src 'self';`中心とし、信頼されたホスト以外を遮断する。  
- **Remote API禁止**: アプリから外部ネットワークにアクセスする機能は持たせない（`@tauri-apps/plugin-http`等は未使用）。将来AI連携で外部通信が必要になった際は別途要件化し、送受信許可ホストを厳格に制限する。  
- **任意埋め込みiframe禁止**: 外部サイト埋め込みやWebView内iframeで未知のページを表示しない。必要な場合も、自己責任の上で事前定義された安全なものに限る。  
- **Shellプラグイン制限**: `tauri-plugin-shell` の `open` コマンドは、少なくとも v2.2.1 以上でのみ使用し、許可するプロトコル（http/https/mailto等）をホワイトリスト化。未信頼入力での任意コマンド実行は不可【4†L184-L190】。  
- **External Launcher禁止**: OSコマンドや任意アプリの起動は原則禁止。必要な場合は`opener`のみを使用し、その場合も入力値を厳しく検証する（リンクも外部サイトではなくSuno/Udioの既知URLに限定等）。  
- **コマンドスコープ制御**: Rust側で定義するコマンド（invoke）は、全ウィンドウから使える設計にはせず、必要最小限のスコープに限定する。`tauri.conf.json`で使用するコマンドを明示的に列挙し、余計なコマンド公開は行わない。  
- **権限スコープの最小化**: `tauri.conf.json` の`capabilities`設定では、真に必要な機能のみ有効化。例えばファイルシステムはプロジェクト専用フォルダに限定し、広域のfs権限（`fs:default`など）は付与しない【26†L12-L14】。メインウィンドウには1つのセキュリティレベル(capability)のみ与える。  
- **Devサーバー利用ルール**: Tauri内蔵の開発用HTTPサーバー（`pnpm tauri dev`等）は、**安全が保証されたネットワーク**（社内LANやVPN経由）でのみ使用し、不特定環境での利用は避ける。  

これらにより、**「外部コード不実行」「必要最小限の権限設定」「ローカル完結」**を徹底します【4†L184-L190】【26†L12-L14】。

## セキュリティリスクと緩和策

- **XSS・コードインジェクション**: TauriアプリはWebView上で動作するため、ユーザー入力や外部断片にスクリプトが含まれていないか要注意です。CSPを厳格にし、未知ソースの読み込みを遮断します【4†L184-L190】。また、プラグインベースの権限管理により、ファイル操作等は明示的に許可済みのものだけに限定し、XSSリスクを低減します【26†L12-L14】。  
- **Electron脆弱性との比較**: Electronと異なり、TauriはOS提供のWebView更新による最新ブラウザ機能を活用します【26†L9-L14】。ElectronはChromiumをバンドルして更新が遅れるためセキュリティ面で不利ですが、TauriはOS更新に追従しており、また細かいファイルアクセス制御が可能です。  
- **外部依存への注意**: 公式で「CDNのスクリプトは攻撃ベクトルになりうる」とされているため【4†L184-L190】、ライブラリは全てローカルにバンドルし、CSPとハッシュ/nonce機構で正当性を担保します。  
- **Dev環境の安全性**: 開発時に使うTauri devサーバーは相互認証・暗号化がないため、VPN等で隔離した環境で運用し、社外ネットワークでの稼働を禁じます【4†L184-L190】。  
- **SQLiteデータの保護**: 歌詞データベースはSQLite(ローカルファイル)を想定しますが、権限を最小化し内部のみでアクセスします。外部からのSQL挿入防止にはSQLiteプラグインのプリペアドステートメントを活用します【21†L200-L204】。  

これらの対策でリスクを最小限に抑え、安全に運用できる設計とします。

## MVP画面構成

MVPでは以下の**3画面/3ペイン構成**を想定します。画面遷移は主に右上メニュー等で切り替えられます。

1. **歌詞編集画面**  
   - *左ペイン*: プロジェクト/歌詞版ツリーを表示するナビゲーション。  
   - *中央ペイン*: Monacoエディタで歌詞本文を編集。選択行へのコンテキストメニューから差分表示やコピーが可能。  
   - *右ペイン*: 選択歌詞版のメタ情報（作成日時、メモ、タグなど）と、コピー/スナップショットボタンなど。  

2. **差分確認画面**  
   - *左ペイン*: 歌詞版リスト。比較元・比較先の版を選択するUI。  
   - *中央ペイン*: MonacoのDiffエディタで差分を表示。追加・削除行が色分けされる。  
   - *右ペイン*: 差分のサマリー（変更箇所数、凡例など）や、必要なら「この差分でリセット/更新」のボタン。  

3. **曲紐付け画面**  
   - *左ペイン*: 現在プロジェクト内の歌詞版一覧と、生成済み曲リスト。  
   - *中央ペイン*: 新規曲登録・既存曲情報表示用エリア。Suno/Udio URL入力フォームやファイル選択ボタンを配置。  
   - *右ペイン*: 選択中の曲の詳細（曲名、生成日時、評価メモ）と、「歌詞版との紐付け」情報。  

画面遷移の簡易ワイヤーをMermaidで示します：

```mermaid
flowchart TB
  subgraph 編集画面
    A1[プロジェクト一覧 (左ペイン)] --> B1[歌詞エディタ (中央ペイン)]
    B1 --> C1[メタ情報/コピー (右ペイン)]
  end
  subgraph 差分画面
    A2[バージョンリスト (左)] --> B2[Diffエディタ (中央)]
    B2 --> C2[差分サマリ (右)]
  end
  subgraph 紐付け画面
    A3[プロジェクト/曲リスト (左)] --> B3[曲登録UI (中央)]
    B3 --> C3[曲詳細 (右)]
  end
```

各画面の機能はMVP時点で上記が基本セットです。UIフレームワーク（React/Vue）のレイアウトコンポーネントで簡易に実装でき、Monacoの機能を中央ペインに自在に配置できます。

- **Monaco組込**: Diff機能含むため設計検証が必要。  
- **データ永続化**: 歌詞や履歴量が増えるとSQLiteパフォーマンス検討要。  
- **バージョン管理**: 差分算法・UI整合が複雑。変更点マージなど追加要望リスクあり。  
- **UI/UX**: 新規UIなのでユーザビリティ検証必須。座学だけでは抜けや勘違いが生じやすい。  
- **セキュリティ設定**: Tauri設定ミスによる脆弱性リスク。CSP漏れや権限過剰付与を防ぐレビューが必要。

## 次のアクションプラン

**短期 (1ヶ月以内)**

1. **ドメインモデルと画面ワイヤ設計の確定**: 上記表やワイヤをレビューし、ステークホルダーと要件の齟齬がないか確認する。  
2. **プロトタイプ作成 (Monaco + Tauri)**: Monacoエディタを組み込んだTauriアプリを起動し、歌詞入力と簡易コピーが動作するか検証する。  
3. **SQLiteプラグイン設定**: `tauri-plugin-sqlite` を追加し、基本的なテーブル作成と保存・取得を試験。歌詞バージョンの保存・ロード機能の基盤を構築する。

**中期 (1〜3ヶ月)**

1. **履歴管理機能の実装**: 歌詞版の保存・ロード・差分表示機能を実装。バージョンツリーのUIも組み込み、改稿前後を遡れるようにする。  
2. **曲紐付けUIの実装**: Suno/UdioのURLや生成ファイルを登録するUIとデータ保存ロジックを追加。曲と歌詞版の対応関係を確立し、ビューに表示できるようにする。  
3. **セキュリティ設定の検証**: CSPやcapabilities設定をテスト。外部スクリプト読み込みの無効化、ファイルアクセス権限の最小化などをコードレビューし、安全性を担保する。  

これらを順次実施し、最低限のMVPが完成した段階で、ユーザーテストとフィードバックを行います。その後、必要に応じてAI作詞補助機能やスマホ連携機能を検討・追加していきます。
