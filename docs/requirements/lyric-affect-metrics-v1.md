# 歌詞感情メトリクス要件 v1

## 1. 目的

歌詞本文の感情傾向と表現密度を、作詞中に参照できる数値として表示する。  
目的は採点や矯正ではなく、歌詞の温度、緊張、余白、詰まり具合を作り手が把握するための補助指標を増やすことである。

## 2. 背景

`affect-wave` は会話テキストから `top_emotions`、`trend`、`wave_parameter` を推定する。  
LyricLytic では外部サービスや別サーバーへの依存を増やさず、同じ出力思想の一部を歌詞編集向けに軽量化して取り込む。

## 3. スコープ

### 3.1 含めるもの

- 現在表示中の歌詞テキストを対象にした同期的な数値算出
- 上位感情ラベル 3 件とスコア
- `valence`、`arousal`、`stability` の trend 値
- `density` を含む wave parameter 相当の補助値
- 行数、文字数、語彙多様度、行密度の表示
- 日本語と英語の最低限の感情マーカー

### 3.2 含めないもの

- 外部 API 呼び出し
- `affect-wave` サーバーへの自動接続
- 埋め込みモデル依存の本格感情推定
- 品質点、合否、禁止語判定
- 自動修正や自動置換

## 4. 機能要件

- 歌詞編集画面の右ペインに `感情メトリクス` を表示できること。
- 空テキストでは中立値を返し、エラー表示にしないこと。
- 感情ラベルは `joy`、`calm`、`curiosity`、`surprise`、`tension`、`sadness`、`anger`、`fear` の 8 種を扱うこと。
- `top_emotions` は常に 3 件をスコア降順で返すこと。
- `trend.valence` は `-1.0..1.0`、その他の数値は `0.0..1.0` に収めること。
- `density` は歌詞の詰まり具合、行密度、語彙多様度、感情の同時活性を合わせた指標として扱うこと。
- 表示文言では、スコアが創作品質の真理値ではないことを明示すること。

## 5. 非機能要件

- 通常の本文入力を阻害しないよう、解析はフロントエンド内の軽量同期処理とする。
- 永続化対象にしない。保存データや履歴構造を変更しない。
- ローカル完結方針を守る。
- 計算式はテスト可能な純粋関数として分離する。

## 6. 受け入れ基準

- 日本語の明るい歌詞で `joy` または `calm` が上位に出る。
- 不安や恐怖を含む歌詞で `fear`、`tension`、`sadness` のいずれかが上位に出る。
- 感情マーカーが多い歌詞は中立テキストより `arousal` または `density` が高くなる。
- 空テキストでクラッシュせず、トップ感情 3 件と全メトリクスを返す。
- Vitest で解析ロジックの値域と代表ケースを検証できる。
