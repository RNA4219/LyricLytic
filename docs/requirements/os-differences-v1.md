# LyricLytic Windows / macOS 差分要件 v1

## 1. 目的

本書は、LyricLytic MVP を Windows と macOS で提供する際に、実装差分が発生しやすい項目を定義する。  
目的は、OS ごとの差異で UX や保存モデルが破綻しないよう、MVP の既定挙動を固定することにある。

## 2. 基本方針

- ドメインモデルと保存形式は Windows / macOS で共通とする
- SQLite スキーマ、エクスポート形式、論理削除モデルは OS 非依存とする
- OS 差分は UI ショートカット、ファイルダイアログ、パス表記、外部 URL オープン確認などに限定する
- OS ごとの見た目よりも、挙動の一貫性を優先する

### 2.1 PoC 実装方針

PoC では、OS ごとのネイティブ最適化を深追いせず、以下を優先する。

- 共通コードで成立する挙動を優先する
- 差分はショートカット表記とダイアログ利用に限定する
- ネイティブメニュー統合や Finder / Explorer 連携の最適化は必須にしない
- まずは `Windows 11` と `macOS 最新安定版` で主要フローが通ることを目標とする

## 3. 差分管理対象

MVP で差分を明文化する対象は以下とする。

1. キーボードショートカット
2. ファイル / フォルダ選択ダイアログ
3. パス表記と表示
4. エクスポート出力先の既定候補
5. 外部 URL オープン確認
6. アプリメニュー位置

## 4. 共通要件

- Project データ構造は OS によらず同一であること
- `.lyrlytic.zip` の生成内容は OS によらず同一であること
- `manifest.targetOs` には `["windows","macos"]` を含めること
- URL、ローカルパス、日時は内部表現を統一すること

## 5. キーボードショートカット

### 5.1 基本方針

主要ショートカットは、Windows では `Ctrl`、macOS では `Command` を使う。

### 5.2 MVP で固定する主要操作

- 保存: `Ctrl+S` / `Command+S`
- 検索: `Ctrl+F` / `Command+F`
- コピー: `Ctrl+C` / `Command+C`
- 差分確認を開く: `Ctrl+Shift+D` / `Command+Shift+D`
- 曲紐付けを開く: `Ctrl+Shift+L` / `Command+Shift+L`
- エクスポートを開く: `Ctrl+Shift+E` / `Command+Shift+E`
- インポートを開く: `Ctrl+Shift+I` / `Command+Shift+I`
- 削除済みデータ管理を開く: `Ctrl+Shift+R` / `Command+Shift+R`

### 5.3 表示要件

- UI 上のショートカット表記は OS に応じて出し分けること
- ドキュメント内でも `Ctrl / Command` 併記を基本とすること

## 6. ファイル / フォルダダイアログ

### 6.1 共通方針

- インポート、エクスポート、ローカルファイル参照は OS ネイティブダイアログを利用する
- アプリ独自の擬似ファイルブラウザを MVP では持たない

### 6.2 インポート

- `.txt` のみ選択可能なフィルタを既定とする
- 文字コード判定に失敗した場合は、再選択導線を出す

### 6.3 エクスポート

- PoC では `保存ダイアログ` を採用し、最終生成ファイル名はアプリ側で初期入力する
- ユーザーは保存場所とファイル名を変更できること
- 保存ダイアログの既定拡張子は `.lyrlytic.zip` とする

## 7. パス表記

### 7.1 内部保持

- DB や export JSON に保存するローカルパスは、OS 上で得られた絶対パス文字列をそのまま保持する
- 正規化のために他 OS 形式へ変換しない
- PoC ではパス正規化ライブラリによる他 OS 互換変換を行わない

### 7.2 UI 表示

- UI 上は OS ネイティブの区切り文字で表示してよい
- export JSON は保持値をそのまま出力する

### 7.3 制約

- ローカルパスを他 OS へ移行可能である前提にしない
- SongArtifact の `source_file_path` は参照情報であり、可搬性保証の対象外とする

## 8. エクスポート出力先

### 8.1 既定候補

- Windows: ユーザーの Documents フォルダ配下を既定候補にしてよい
- macOS: ユーザーの Documents フォルダ配下を既定候補にしてよい

PoC では両 OS ともに `Documents/LyricLytic Exports/` を既定候補とし、存在しない場合はアプリ側で作成を試みてよい。

### 8.2 要件

- 出力先はユーザーが変更できること
- 出力完了後、OS 依存の方法で保存先を開くことは必須にしない
- 生成ファイルパスの表示だけで MVP は成立する
- PoC では `保存先を開く` ボタンは任意実装とし、未実装でもよい

## 9. 外部 URL オープン

### 9.1 基本方針

- URL は参照情報として保持する
- 自動オープンはしない
- ユーザーが明示操作した時のみ既定ブラウザで開く

### 9.2 確認

- Windows / macOS ともに、外部 URL を開く前に確認 UI を出してよい
- 少なくとも未検証 URL を無言で開かないこと
- PoC では常に確認ダイアログを出す固定挙動でよい
- 確認ダイアログには URL 全文、`既定ブラウザで開く`、`キャンセル` を含めること

## 10. アプリメニュー

### 10.1 基本方針

- Windows はウィンドウ内メニューまたはタイトルバー近傍導線を許容する
- macOS はアプリメニューバー統合を考慮する

### 10.2 MVP

- 主要導線が画面内から到達できることを優先し、ネイティブメニュー統合は必須にしない

### 10.3 PoC 既定

- PoC では、主要操作をすべて画面内ボタンまたはショートカットから到達可能にする
- macOS 専用メニューバー最適化は後回しとする
- Windows 専用コンテキストメニュー最適化も後回しとする

## 11. 検証対象

PoC で最低限通すべき検証は以下。

1. Windows で `.txt` インポートが成功する
2. macOS で `.txt` インポートが成功する
3. Windows でエクスポート zip が生成される
4. macOS でエクスポート zip が生成される
5. 両 OS でクリップボードコピーが成功する
6. 両 OS で外部 URL 確認ダイアログが表示される
7. 両 OS で主要ショートカットが期待通り動作する

## 12. 仕様化で見つかった論点

1. ショートカット表記は OS 切替が必要
2. パスは内部正規化しすぎると可読性や参照性が壊れる
3. ローカルファイル参照は OS 間可搬性を保証できない
4. エクスポート保存先を自動で開く要件は OS 差分を増やすので MVP では必須にしない
5. ネイティブメニュー統合を必須にすると macOS 側の工数が膨らみやすい

## 13. 要件へ戻すべき項目

- ショートカットは `Ctrl / Command` で OS 別に出し分けること
- OS ネイティブダイアログを使うこと
- SongArtifact の file path は可搬性保証外であること
- 生成ファイルパス表示で MVP を満たすこと
- URL は明示操作時のみ外部ブラウザで開くこと
- PoC 検証対象 OS を Windows 11 / macOS 最新安定版に置くこと
- PoC の保存ダイアログ方式を固定すること

## 14. Linux 将来対応メモ

Linux は MVP / PoC の必須対象外とし、将来対応時は以下を制約メモとして扱う。

1. 主要対象ディストリビューションは Ubuntu LTS 系を優先候補とする
2. ショートカット表記は `Ctrl` 基準とする
3. ファイルダイアログ挙動、保存先既定候補、既定ブラウザ起動の差分は別途検証前提とする
4. SongArtifact のローカルパス可搬性は Windows / macOS と同様に保証しない
5. Linux 対応は既存ドメインモデルと export 形式を変更せず、UI / packaging 差分の吸収で対応する

このメモは将来対応時の出発点であり、現行 MVP / PoC 着手判断の未確定事項とはしない。

## 15. 実装済み OS 対応

### 15.1 デフォルト設定の OS 非依存化 (2026-04-01)

LLM 設定のデフォルト `modelPath` を Windows 固有パスから空文字に変更した。
これにより、macOS ユーザーが不適切なパスを設定される問題を回避した。

**変更内容:**
- `LLMSettingsPanel.tsx`: デフォルト `modelPath` を空文字に変更
- `Editor.tsx`: デフォルト `modelPath` を空文字に変更
- プレースホルダを `/path/to/your/models (optional)` に変更

**ユーザー対応:**
- LLM 機能を使用する場合、ユーザーが自身の環境に合わせてモデルパスを設定する必要がある
- 設定画面でモデルパスを明示的に入力する導線を提供
