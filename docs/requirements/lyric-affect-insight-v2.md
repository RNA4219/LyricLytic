# 歌詞感情インサイト要件 v2

## 1. 目的

歌詞全体の平均値だけでなく、曲構成の中で感情と密度がどう変化しているかを、作詞中に判断できる粒度で可視化する。  
目的は採点ではなく、Verse / Pre / Chorus / Bridge などの役割ごとに、温度、緊張、余白、詰まり具合、変更差分を制作判断へつなげることである。

## 2. スコープ

### 2.1 含めるもの

- セクション別の感情メトリクス
- 行単位またはセクション単位の感情波形
- 保存済みスナップショット同士の感情・密度差分
- 感情スコアに寄与した語・行・表現の根拠表示
- 制作向けアラート
- ゴールデン歌詞セットによる回帰テスト

### 2.2 含めないもの

- 外部 API、別プロセス、埋め込みモデルへの必須依存
- 創作品質の合否判定
- 自動置換、自動修正、強制的な改善指示
- 保存データ構造や SQLite スキーマの変更

## 3. 機能要件

### 3.1 セクション別メトリクス

- `Verse`、`Pre`、`Pre-Chorus`、`Chorus`、`Bridge`、`Outro` などのセクションごとに、上位感情、`valence`、`arousal`、`density`、`tension` を算出できること。
- セクション名は既存の `Section.displayName` / `Section.type` を使用すること。
- セクション本文が空の場合はクラッシュせず、中立値を表示すること。
- 全文メトリクスとセクション別メトリクスを同じパネル内で比較できること。

### 3.2 感情波形ビュー

- 行またはセクションの順番に沿って、`valence`、`arousal`、`density`、`tension` を並べて表示できること。
- 値域は `valence = -1.0..1.0`、その他は `0.0..1.0` に収めること。
- 曲中のピークや平坦さを視覚的に把握できること。

### 3.3 スナップショット比較

- DiffViewer 上で左右に選択した保存済みバージョンの歌詞を解析し、`valence`、`arousal`、`density`、`tension` の差分を表示できること。
- 差分は「明るさが上がった」「密度が下がった」「緊張が増えた」など、制作判断に読める短い文で補足すること。
- 比較対象のどちらかが未選択の場合は、比較結果を表示しないこと。

### 3.4 根拠表示

- 上位感情や `tension` / `sadness` などに寄与した語、行、表現を最大数件表示すること。
- 根拠は該当感情、該当行、ヒットした語句、寄与スコアを含むこと。
- 数値のブラックボックス化を避けるため、少なくとも全文メトリクスには根拠表示を含めること。

### 3.5 制作向けアラート

- 次の観点を参考情報として提示できること。
  - Chorus の密度が Verse より低い
  - 終盤まで感情変化が平坦
  - 英日混在により syllable / density 推定が弱い可能性がある
  - 高緊張または低明度のセクションが連続している
- アラートは警告ではなく制作メモとして扱い、保存や編集を阻害しないこと。

### 3.6 ゴールデン歌詞セット

- 明るい歌詞、暗い歌詞、高密度ラップ調、低密度バラード調を固定サンプルとして持つこと。
- ゴールデンセットに対して、主要感情、密度、熱量、行波形の基本挙動を Vitest で検証できること。
- 将来の辞書・重み調整で意図しない退行を検知できること。

## 4. 非機能要件

- 解析はフロントエンド内の軽量な純粋関数として実装する。
- UI 表示は既存の 3 ペイン構造を崩さない。
- 長文入力でも通常編集を阻害しないよう、React 側では `useMemo` で再計算範囲を限定する。
- 既存の保存データ、スナップショット、論理削除、DiffViewer の契約を変更しない。

## 5. 受け入れ基準

- 右ペインで全文メトリクス、セクション別メトリクス、感情波形、根拠、制作アラートを確認できる。
- DiffViewer で左右の保存済みバージョンを選ぶと感情・密度差分が表示される。
- ゴールデン歌詞セットを使った Vitest が通る。
- `npm test -- --run` と `npm run build` が通る。
- five-tool-validation-gate の証跡として、要件、静的解析、自動テスト、手動 BB 観点、Gate 記録の保存先が確認できる。
