# LyricLytic フロント質感差分レビュー

更新日: 2026-04-01

## 1. 目的

本書は、要件定義段階で作成した WebUI モックと、現行実装のランタイム UI の `質感` の相違を整理するためのレビュー文書である。  
ここでいう質感には、色、余白、タイポグラフィ、情報密度、文言トーン、操作面の見え方を含む。

比較元:

- モック正本: `stitch/lyric_editor_working_draft_with_char_counts/code.html`
- モック参考画像: `stitch/lyric_editor_working_draft_with_char_counts/screen.png`
- 現行実装: `src/pages/Editor.tsx`
- 現行スタイル: `src/styles/editor/workspace.css`, `src/styles/editor/section-tabs.css`
- 実行時参考画像: `docs/runtime-captures/lyriclytic-runtime-screen-20260401.png`

## 2. 総評

現行実装は `3 ペイン構成` と `Working Draft 主体` という構造面では要件へ寄っている。  
一方で質感は、モックが目指していた

- `Digital Atelier` 的な静かな高級感
- 情報を絞ったインスペクタ
- エディタを主役にした余白の大きい構図

から離れ、現在は

- 業務 UI に近い密度
- 機能ボタンが並ぶユーティリティパネル
- 見出しや文言が説明的すぎる状態

へ寄っている。

## 3. 相違点一覧

### GAP-V01 タイポグラフィの格が落ちている

- モック:
  - 中央の見出しにセリフ体相当の大きなタイトルを使い、本文も文学的な読み味を優先していた
  - 例: `Verse 1` の大見出し、本文の長い行長、控えめな補助ラベル
- 現行:
  - `workspace.css` の `editor-pane-header h2` は `1.125rem` で小さく、Monaco 本文も実務エディタ寄り
  - `Editor.tsx` でも見出しは `Working Draft` と説明文中心になっている
- 影響:
  - 作詞 IDE というより、設定と編集を並べた管理 UI に見える

### GAP-V02 左ペインの役割がモックより無骨

- モック:
  - 左ペインは `Project Structure` と進捗、セクション抜粋、Quick Panels を持つ構造ナビだった
- 現行:
  - `VersionPane.tsx` は `📚 バージョン`, `保存中...`, `バージョン比較`, `プロジェクト削除` など、機能ボタン中心
  - セクション構造は右上へ移り、左は履歴管理寄りになっている
- 影響:
  - モックの「創作ワークスペースの左ナビ」ではなく、「履歴操作のサイドバー」に見える

### GAP-V03 中央ペイン上部が説明過多で、主役感が弱い

- モック:
  - 中央は `Verse 1`, `Working Draft`, `Autosaved 2m ago` の最小情報で、本文を主役にしていた
- 現行:
  - `Editor.tsx` の `editor-pane-hint` が長く、`全文をまとめて確認できます。...` の説明が常時表示される
  - `workspace.css` でもヘッダ領域が UI 説明用に割かれている
- 影響:
  - 初見には親切だが、常用時の没入感を削っている

### GAP-V04 右ペインが「インスペクタ」ではなく「アクション倉庫」になっている

- モック:
  - 右ペインは `Search / Fragments / Song / History` のタブで情報を絞って見せる構成
- 現行:
  - `ActionPane.tsx` は `スナップショット保存`, `エクスポート`, `インポート`, `検索`, `断片`, `曲リンク`, `LLM`, `StyleProfile`, `プロジェクト設定` が縦に累積
- 影響:
  - 質感が一気に業務アプリ化し、モックの静かな整理感が消えている

### GAP-V05 文言トーンが混在している

- モック:
  - `Compare`, `Save Snapshot`, `Suggested Fragments`, `Linked Artifact` など、短くトーンが揃っている
- 現行:
  - `💾 スナップショット保存`, `📥 .txt インポート`, `🎵 曲リンク`, `プロジェクト設定` など、日本語ラベル + 絵文字 + 管理文言が混在
- 影響:
  - ブランドトーンが揃わず、質感のブレとして体感される

### GAP-V06 セクション一覧の情報密度が高く、面の美しさが弱い

- モック:
  - 左ナビのセクション項目は大きめの面で、抜粋は控えめ、アクションは通常隠していた
- 現行:
  - `section-tabs.css` では、番号、入力欄、移動、削除ボタンが常時近接し、カードというより編集行に近い
- 影響:
  - 構造一覧が「読む面」ではなく「操作対象の羅列」に見える

### GAP-V07 追加導線の見え方がモックより荒い

- モック:
  - `Add Section` は 1 つの静かな CTA で、押した後の追加候補は別導線で想像させる作りだった
- 現行:
  - `+Intro`, `+Verse`, `+Pre-Chorus`, `+Chorus`, `+Bridge`, `+Outro`, `+Custom` が常時露出している
- 影響:
  - 便利だが、上質さより操作パレット感が強くなる

### GAP-V08 情報の縦積みが多く、余白で見せられていない

- モック:
  - 中央本文の上下余白が大きく、右ペインのカード間も広めで、面構成で魅せていた
- 現行:
  - `workspace.css` は機能面の詰め込みを優先していて、右ペインやセクション欄が密集しやすい
- 影響:
  - 画面が「便利」には見えるが、「整っている」感じが弱い

### GAP-V09 現行ホーム画面の質感はモック以前の段階に留まっている

- モック:
  - `home_project_selection_updated_background` は背景、カード、Recent Activity、Fragment Library を含む高密度なブランド画面
- 現行:
  - 実行時参考画像では、単純なフォーム中心で、ブランド演出や Recently Updated 的な見せ場がない
- 影響:
  - エディタ以前に、アプリ全体の第一印象がモックと乖離している

### GAP-V10 ブランド補助要素が未実装

- モック:
  - 進捗バー、tempo、fragment card、artifact card、version timeline など、世界観を崩さない補助要素があった
- 現行:
  - 機能は近くても、カード化や timeline 化ではなく、生のフォームやボタン列として出ている
- 影響:
  - 同じ情報でも、完成品ではなく中間実装に見えやすい

## 4. 質感差分の原因

主因は次の 4 点である。

1. モックは `情報を絞る` 方針、現行は `機能を見せ切る` 方針になっている
2. モックは `タイポグラフィと余白` が主役、現行は `フォームとボタン` が主役になっている
3. モックは `タブで文脈を切り替える`、現行は `トグルで全部積む` 方式になっている
4. モックは `英語ベースの短いラベル`、現行は `説明的な日本語 UI` に寄っている

## 5. 先に直すべき差分

優先度順では次の通り。

### P1

- 右ペインを `ActionPane` の積み上げ型から、タブまたは 1 主題表示型へ戻す
- 中央ヘッダの説明文を減らし、本文主役の余白へ戻す
- 左ペインの履歴管理感を弱め、構造ナビ感を強める

### P2

- セクション一覧の操作ボタン常時露出を減らす
- プリセット追加導線を `常時並列` から `1 CTA + 展開` へ戻す
- 文言トーンを絵文字混在から整理する

### P3

- ホーム画面のブランド演出をモックに寄せる
- SongArtifact / Version / Fragment のカード表現を増やす

## 6. 判定

現行実装は `機能の位置` ではモックに近づいているが、`質感` はまだかなり離れている。  
特に右ペイン、見出し階層、文言トーン、余白設計の差が大きい。

一言で言うと、

`モックは創作ワークスペース、現行は機能集約 UI`

という差が残っている。
